# Finance Intelligence Engine

Lightweight local BIST analysis engine built with FastAPI, JSON storage, rule-based analyzers, an optional Gemini explanation layer, Telegram notifications, scheduler-driven execution, history, and performance tracking.

## Installation

1. Create a virtual environment.

```powershell
python -m venv .venv
```

2. Install dependencies.

```powershell
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

3. Create a local environment file from `.env.example`.

```powershell
Copy-Item .env.example .env
```

## Environment Variables

Required:

- `TELEGRAM_BOT_TOKEN=XXXX`
- `TELEGRAM_CHAT_ID=XXXX`

Optional:

- `GEMINI_API_KEY=XXXX`
- `GEMINI_MODEL=gemini-2.5-flash`
- `NEWS_SOURCES=...`
- `TIMEZONE=Europe/Istanbul`
- `STORAGE_DATA_DIR=storage/data`

## Telegram Setup

1. Create a bot with BotFather.
2. Set `TELEGRAM_BOT_TOKEN` in `.env`.
3. Get your target chat id and set `TELEGRAM_CHAT_ID`.
4. Start the app and run the daily pipeline or wait for the scheduler.

## Gemini Setup

1. Set `GEMINI_API_KEY=XXXX` to your real local value.
2. Keep `GEMINI_MODEL=gemini-2.5-flash` or replace it with another supported Gemini model.
3. If Gemini is unavailable, the project continues normally and only skips AI summaries.

## Scheduler

Weekday automated jobs in `Europe/Istanbul`:

- `09:55`: full pipeline run
   - collect news
   - collect KAP
   - collect market
   - analyze news
   - analyze market
   - aggregate ticker news
   - generate recommendations
   - optional Gemini summary
   - Telegram notification
   - archive history
   - update performance snapshot
- `18:10`: end-of-day closeout
   - read active recommendations
   - read latest market snapshot
   - close daily entries
   - update `history.json`
   - update `performance.json`

## JSON Files

- `storage/data/news.json`: normalized multi-source news
- `storage/data/kap.json`: KAP announcements
- `storage/data/market.json`: latest market snapshot
- `storage/data/analysis.json`: aggregate daily analysis
- `storage/data/news_analysis.json`: rule-based news intelligence output
- `storage/data/market_analysis.json`: technical market intelligence output
- `storage/data/ticker_news_summary.json`: aggregated ticker-news mapping
- `storage/data/recommendations.json`: top 10 daily recommendations
- `storage/data/history.json`: archived daily signals and closed trade outcomes
- `storage/data/performance.json`: aggregate trade performance summary

## Daily Workflow

1. Scheduler starts the daily pipeline at 09:55 on weekdays.
2. Recommendations are generated by the rule-based engine.
3. Gemini optionally explains the output.
4. Telegram sends the daily recommendation list.
5. Recommendations are archived into history.
6. At 18:10, the day is closed and performance is refreshed.

## Troubleshooting

- If `Start Project` fails, run `.\run.bat` directly.
- If Telegram is not configured, recommendations are still generated and archived.
- If Gemini fails, the application continues without AI summaries.
- If market movement is flat outside trading hours, use the latest available market snapshot.
- To auto-start on Windows logon, import `finance-startup-task.xml` in Windows Task Scheduler and replace `C:\PATH\TO\finance` with your local project path.
